| 2 | 8px (0.5rem) | Pill-chip horizontal padding (`px-2`), tight rows |
| 3 | 12px (0.75rem) | Sidebar nav item vertical padding (`py-3`) |

| 6 | 24px (1.5rem) | About description line-block; social-icon margin-right (recovered `1.5rem`); `.subheading` font-size baseline |

| 20 | 80px (5rem) | `.resume-section` vertical padding (recovered: `5rem` top + `5rem` bottom) |

**Phase-specific exceptions:**

- **Sidebar width (desktop, ≥ 768px)** = `19rem` (304px) — recovered from CRA `#sideNav { width: 19rem }`. Not on the 4-step scale; load-bearing for visual parity per CONTEXT.md D-01.
- **Body padding-left (desktop, ≥ 768px)** = `17rem` (272px) — recovered from CRA `body { padding-left: 17rem }` at `@media (min-width: 992px)`. Phase 3 applies this at `md` per CONTEXT.md D-10 (intentional widening from `lg`→`md`).
- **Mobile top bar padding-top** = `3.375rem` (54px) — recovered from CRA `body { padding-top: 3.375rem }`. Not on the 4-step scale; matches existing CRA value.

- **`.resume-section` min-height** = `100vh` at `≥ 768px` — recovered; means each section is at least one viewport tall (visual parity).
- **`.profile-pic` dimensions** = `200px × 225px` (mobile About section, < md only per D-25).
- **Sidebar profile photo max-width/max-height** = `20rem × 20rem` (desktop sidebar, ≥ md).

## Typography

Phase 3 declares **exactly 4 size roles** (Body, Lead, Subheading, Heading/Display via `.subheading` + `<h1>`) and **exactly 2 weight pairings** (Mulish 400 body / 800 strong + Saira 500 subheading / 700 heading). All values recovered from `git show b537845:src/App.css` (CRA pre-wipe). Italics in Mulish (400i, 800i) are loaded but not introduced as new typography tokens — they exist solely for the testimonial `<blockquote>` styling per CONTEXT.md D-24.

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body | 16px (1rem) | 400 (Mulish regular) | 1.5 | `--font-body` (Mulish) |
| Lead (About description, project description) | 18.4px (1.15rem) — `p.lead` | 400 (Mulish regular) | 1.5 | `--font-body` (Mulish) |
| Subheading (h2/h3 + `.subheading`) | 24px (1.5rem) — `.subheading` baseline; per-section heading sizes follow Tailwind defaults (`text-xl`/`text-2xl`) where the recovered CSS did not pin them | 500 (Saira Extra Condensed medium) | 1.2 | `--font-heading` (Saira), `text-transform: uppercase`, `letter-spacing: 0.05rem` |
| Heading / Display (h1 — name in About) | Tailwind `text-6xl` (60px) — Phase 3 planner judgment; recovered CSS sets `h1 { line-height: 1 }` but does not pin `font-size`; CRA Bootstrap default for h1 was 2.5rem (40px) but visual parity at the existing site shows the name renders much larger via Bootstrap `display-4` (3.5rem). Phase 3 lands at `text-6xl` to preserve "name dominates above-fold" visual weight. Adjustable to `text-5xl` if recovered CSS audit during execution surfaces a smaller value. | 700 (Saira Extra Condensed bold) | 1 (recovered: `h1 { line-height: 1 }`) | `--font-heading` (Saira), `text-transform: uppercase` |

**Italics:** Mulish 400i used for `<blockquote>` testimonial text (CONTEXT.md D-24); Mulish 800i loaded for parity with locked CRA weight set (`Muli:400,400i,800,800i`) but not introduced as a separate typography role.

**Sidebar nav links:** Saira-equivalent uppercase via existing `--font-heading`; `font-weight: 800` (Mulish-equivalent — but recovered CRA uses Saira pattern via `#sideNav .nav-link` block at weight 800 with `letter-spacing: 0.05rem` and `text-transform: uppercase`). Active state adds bold visual via accent color + left border (CONTEXT.md D-12). Inactive sidebar links use the same 800 weight + uppercase + tracking — the active state is conveyed by color + border, not weight delta (matches recovered CRA pattern). Planner judgment: this is acceptable because the active state still satisfies "more than color alone" via the left border (D-12).

## Color

Recovered from CRA `git show b537845:src/App.css`. The site's effective palette is **1 dominant cream + 1 secondary text-gray + 1 single accent rust/orange + 1 link-navy**. The 60/30/10 split below is the natural derivation; no second semantic color (no destructive actions exist in Phase 3 — see Copywriting Contract below).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#eee2dc` (cream/off-white) | Body background, sidebar surface, card surface, mobile top-bar surface — **all surfaces sit on the same cream per CONTEXT.md D-04 (no white card-shadow treatment)** |
| Secondary (30%) | `#6c757d` (Bootstrap gray) | All body paragraph text, secondary text in cards, sidebar inactive nav-link text, `<cite>` attribution, education/work date columns, `.dev-icons` label captions |
| Accent (10%) | `#bd5d38` (warm rust/orange) | Reserved-for list below — never "all interactive elements" |
| Link default | `#123c69` (deep navy) | "View on GitHub →" link inside project cards; any non-accent `<a>` (email link in About contact CTA) |
| Link hover | `#123c69ad` (deep navy at ~68% alpha) | Hover state on `#123c69` links |
| Accent focus tint (optional) | `#d48a6e` (accent softer tint) | Mobile hamburger focus ring (planner may use the accent itself `#bd5d38` for stronger contrast) |
| Social-icon hover bg | `#8d9ba8` (cool gray-blue) | Social-icon circle hover state in About section |
| `<hr>` border-color | `currentColor` at ~10% alpha (suggested `border-black/10` Tailwind utility) | Section dividers per CONTEXT.md D-03 — Tailwind v4 + Pitfall 14 requires explicit border-color or the `<hr>` picks up `currentColor` solid |
| Destructive | n/a | **No destructive actions in Phase 3 — see Copywriting Contract** |

**Accent reserved for:**

1. The `<span class="text-primary">Panchani</span>` highlight in the `<h1>` of About (existing CRA pattern — last-name accent).
2. Sidebar nav-link **active** state: text color `#bd5d38` + 3–4px left border `#bd5d38` + 800 weight (already inherited) — 3 signals per CONTEXT.md D-12 (satisfies WCAG 1.4.1 / A11Y-04).
3. Sidebar nav-link **hover** state (text color shifts to `#bd5d38`).
4. Section heading underline / accent treatment if the recovered CRA pattern uses one (planner verifies during execution).
5. Resume Download button background per CONTEXT.md D-27 (visible label "Download Resume").
6. Project card title hover state (the `<h3><a href={project.data.url}>` per CONTEXT.md D-18 stretched-link pattern).
7. "View on GitHub →" link hover state inside project cards (transition from `#123c69` to `#bd5d38`).

**Accent NOT reserved for:**

- Body paragraph text (uses `#6c757d`).
- Pill-chip backgrounds for tech-stack (use `bg-black/5` neutral per RESEARCH.md "Pill-chip styling" Claude's Discretion).
- Inactive sidebar nav links (use `#6c757d`).
- `<cite>` attribution under testimonials (use `#6c757d`).
- Section dividers (use `border-black/10`).

**Contrast notes (Phase 4 owns formal axe-core verification):**

- `#6c757d` text on `#eee2dc` background ≈ 4.0:1 — **below WCAG AA 4.5:1 for normal text**. - `#bd5d38` accent on `#eee2dc` background ≈ 4.6:1 — passes AA for normal text.
- `#123c69` link on `#eee2dc` background ≈ 9.5:1 — passes AA + AAA.

## Interaction Contract (Phase 3 specific)

This section is not in the standard UI-SPEC template but is critical for Phase 3 because it has more interaction surface than typical render-only phases. Documented here so the executor has a single source.

| Interaction | Trigger | Behavior | Source |
|-------------|---------|----------|--------|
| Nav link click | User clicks `<a href="#projects">` etc. | Native browser anchor navigation; CSS `html { scroll-behavior: smooth }` provides the easing | CONTEXT.md D-32 |
| Reduced motion | `@media (prefers-reduced-motion: reduce)` matches | Override `html { scroll-behavior: auto }` — no smooth scroll | CONTEXT.md D-32; NAV-02 |
| Scroll-spy | User scrolls main content | `IntersectionObserver` on `document.querySelectorAll('main section[id]')` with `rootMargin: '-30% 0px -50% 0px'`, `threshold: 0`; on intersection change, set `aria-current="page"` on matching `nav a[href="#<id>"]`, remove from others | CONTEXT.md D-30, D-31; NAV-03 |
| Mobile hamburger toggle | User clicks/taps/Enter/Space on `<button aria-controls="sidenav-list">` | Toggle `aria-expanded` between `true`/`false`; toggle visibility of `#sidenav-list` (hidden class) | CONTEXT.md D-13; NAV-04 |
| Mobile nav Escape | User presses Escape while sidenav-list is open | Close the list (set `aria-expanded="false"`, hide); keep focus on the hamburger button | CONTEXT.md D-13 |
| Project card whole-card click | User clicks anywhere on the project card | Navigate to `project.data.url` (new tab) — implemented via CSS "stretched link" `::after { position: absolute; inset: 0 }` on the card title's `<a>` (no nested anchors, no JS click handler on the card div) | CONTEXT.md D-18 |
| External link new-tab | User clicks any link with `target="_blank"` | Browser opens new tab; visually-hidden "(opens in new tab)" announces to AT; `rel="noopener noreferrer"` on every external `<a>` | CONTEXT.md D-18, D-26 |
| Resume download | User clicks "Download Resume" button | `<a href="/Rashmil_Panchani.pdf" download>` triggers browser file download | CONTEXT.md D-27; SECTION-02 |
| Focus rings | User tabs through interactive elements | Tailwind v4 default focus ring preserved; not removed by any Phase 3 CSS | CONTEXT.md D-01 "focus rings honored"; .claude/CLAUDE.md § 5 |
| Active state (sidebar nav) | Section is most-intersecting per observer | 3 signals: text color → `#bd5d38`, 3–4px left border `#bd5d38`, `aria-current="page"` attribute set | CONTEXT.md D-12; WCAG 1.4.1 / A11Y-04 |
